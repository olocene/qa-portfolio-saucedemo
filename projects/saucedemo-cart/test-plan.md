# Test Plan – Cart Functionality (SauceDemo)

## 1. Objective
To verify that cart functionality works correctly, including:
- Adding products
- Removing products
- Badge updates
- Data persistence during navigation

## 2. Scope
### In Scope
- Add to cart
- Remove from cart
- Cart badge updates
- Navigation persistence

### Out of Scope
- Payment process
- Checkout validation
- Backend validation

## 3. Test Items
- Inventory page
- Cart page
- Cart icon (badge)

## 4. Test Strategy
Manual testing based on functional test cases.

## 5. Test Cases Covered
- TC-CART-01: Add product to cart
- TC-CART-02: Remove product from cart
- TC-CART-03: Cart badge updates correctly
- TC-CART-04: Cart persistence during navigation

## 6. Entry Criteria
- Application is accessible
- Test user available

## 7. Exit Criteria
- All test cases executed
- Bugs reported and documented

## 8. Test Environment
- Browser: Chrome (latest version)
- OS: Windows

## 9. Risks
- Session handling inconsistencies
- UI synchronization issues (badge/cart)
